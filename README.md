# rwe-vaccine-effectiveness-synthetic

> A reproducible, end-to-end **real-world evidence (RWE) HPV vaccine-effectiveness
> study** (cancer prevention) built on **fully synthetic** administrative claims
> data — R + SQL (DuckDB), one-command Docker run, tested, and CI-verified.

## Why this repo exists

My production RWE / pharmacoepidemiology work is proprietary and cannot be
published. This project demonstrates the same competencies on data I generate
myself, so **every step is inspectable and runnable**:

- **SQL on a claims schema** — enrollment, medical/pharmacy claims, a
  state-IIS-shaped vaccine registry, mortality — with cohort definition and
  HPV exposure indexing.
- **R analysis** — exposure-indexed time-to-event against an incident high-grade
  cervical lesion (CIN2+) / cervical-cancer outcome (Kaplan–Meier, incidence
  rates, Cox), reported as vaccine effectiveness `VE = (1 − HR) × 100%`.
- **Data quality control** — a deliberately **planted registry anomaly**
  (dose dates that violate HPV product-availability timelines) that the QC step
  detects, quantifies, and explains.
- **Reproducibility** — Docker, `make all`, pinned dependencies, testthat, CI.

> ⚠️ **All data here is synthetic.** It is generated by code in this repo and
> does not describe any real person or any real vaccine's effectiveness. The
> "true" effect is a parameter baked into the generator so the analysis can be
> validated against ground truth.

## Quickstart

> _Pipeline code is not implemented yet — this repo is currently scaffolded.
> See [`PLAN.md`](PLAN.md) for the full build plan. The commands below describe
> the intended one-command run._

```bash
# Run the whole pipeline in a container (DuckDB is embedded — no DB server)
docker compose run --rm analysis make all
```

`make all` will: generate synthetic claims → build the cohort → index exposure →
ascertain outcomes → run the survival analysis → run QC checks → render the
dashboard → run the tests.
